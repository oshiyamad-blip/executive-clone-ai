# 三木谷スタイル・デモ（入力〜完全な出力まで）

実在の著名経営者の思考をどこまで再現できるかを、公開情報ベースのペルソナで検証するデモです。

> ⚠️ **免責**: 本デモのデータは三木谷浩史氏の**公開された経営哲学・著書・公表された意思決定**に基づく
> “再現”であり、本人の非公開情報・発言そのもの・本人/楽天グループ公認のものではありません。
> 評価・検証用途に限定してください。`demo/` の議事録・音声メモは**架空のサンプル**です。

---

## 0. デモDB（何が入っているか）

`DEMO_MODE=true` のとき、Notion不要で以下のペルソナDBが読み込まれます（既定は `mikitani`）。

- **プロファイル**: 価値観（Empowerment / スピード / グローバル …）、意思決定ルール12件
  （スピード最優先、仮説→実行→検証→仕組み化、毎日1%改善、グローバル前提 …）、
  成功/失敗パターン、**権限委譲ライン**
- **シグナル8件**: 英語公用語化、モバイル大型投資、ポイント経済圏、1%改善の複利 …
- **ストーリー2件**: 「英語化が海外展開の土台」「モバイル投資＝短期赤字を許容し経済圏拡張」

切替: `DEMO_PERSONA=mikitani`（既定） / `DEMO_PERSONA=sample`（架空企業）。

## 1. 入力方法：議事録テキスト？ 音声？ → **どちらもOK**

入力は最終的に**テキストファイルを受け皿フォルダに置く**形に統一されます。方法は2つ:

### (A) 会議議事録（テキスト）
議事録を `.txt` / `.md` で書き出し、受け皿へ置く。サンプル: `demo/mikitani-inputs/2026-07-15_meeting-minutes.txt`

### (B) 音声 → 文字起こし（Plaud / Soundcore / スマホ）
録音を文字起こしして `.txt` / `.srt` で受け皿へ置く。サンプル: `demo/mikitani-inputs/2026-07-15_voice-memo.txt`

どちらも次のパイプラインを通ります:
```
ファイル投入 → collect（名寄せ） → extract（重要情報だけシグナル化, Claude）
   → Notion（シグナル/ストーリーDB） → 出力（即断 / 壁打ち / ブリーフィング / ダイジェスト）
```

### フル取り込みを試す（Anthropic＋Notion が必要）
```bash
cp demo/mikitani-inputs/2026-07-15_meeting-minutes.txt lifelog-inbox/
cp demo/mikitani-inputs/2026-07-15_voice-memo.txt      lifelog-inbox/
npm run collect && npm run extract   # 議事録/音声から「提携は小さく検証」「無償は現場裁量」等を抽出
npm run analyze                      # 因果ストーリーを構築
```
> DEMO_MODE ではこの取り込み結果を**あらかじめDBに入れてある**ので、Notionが無くても下記の出力をすぐ試せます。

## 2. 出力を試す（LLM APIキーはどれか1つでOK）

生成には LLM プロバイダのキーが1つ要ります。**Gemini の無料枠**でも Claude でも動きます。

```bash
# Gemini 無料枠（Google AI Studio）を使う場合
printf 'LLM_PROVIDER=gemini\nGEMINI_API_KEY=...\n' >> .env.local
# または Claude を使う場合
echo "ANTHROPIC_API_KEY=sk-ant-..." >> .env.local
```

### 営業/現場の即断
```bash
DEMO_PERSONA=mikitani npm run demo:decide -- "海外SaaSと提携したい。先方が独占契約を求めています。受けていい？"
DEMO_PERSONA=mikitani npm run demo:decide -- "大口顧客が半年の無償提供を要望。応じていい？"
```

### 採用判断
```bash
DEMO_PERSONA=mikitani npm run demo:hire -- --file demo/mikitani-inputs/candidate-sample.txt
# 面接メモや音声書き起こしのテキストをそのまま渡してもOK
DEMO_PERSONA=mikitani npm run demo:hire -- "新規事業開発の中途候補。SIer5年→スタートアップで海外0→1を自走…"
```

### 壁打ち（じっくり）／Web
```bash
npm run demo:web        # http://127.0.0.1:8787 、上部「即断/採用/壁打ち」トグル
npm run demo:chat       # CLI
```
質問例: 「新規事業に参入すべきか、判断軸は？」「競合の世界大手にどう対抗する？」

### 会議前ブリーフィング
```bash
DEMO_PERSONA=mikitani npm run demo:brief -- "モバイル事業への追加投資を続けるべきか"
# → briefings/ にMarkdown生成
```

## 3. 完全な出力イメージ（例）

> 下記は本ペルソナ・プロンプトから想定される**出力例**です（実際の文面は実行時にAIが生成）。

**即断**: 「海外SaaSと提携、先方が独占契約を要望。受けていい？」
```
【結論】要相談（社長確認）
【理由】まず小さく提携して検証し、良ければ仕組み化するのが私の型だ[S6][T1]。
        独占契約は全社方針・長期に関わる大型のコミットで、現場裁量の範囲を超える。
        顧客価値とKPIで見極めるべき段階で、いきなり独占は縛りが強すぎる[S4]。
【権限】社長に確認が必要。
        ただし「まず3か月の“非独占”トライアル＋月次KPI（アクティブ率・単価）で検証」への
        差し戻し提案は、現場の裁量で今すぐ進めてよい。
参照元: [S6] スピード最優先・KPI検証 / [S4] 1%改善の複利 / [T1] 英語化が海外展開の土台
```

**採用判断**: `candidate-sample.txt`（架空: 海外0→1を自走した中途候補）
```
【合否の傾き】採用寄り（確信度: 中）
【評価できる点】
・海外展開を一人で自走し0→月商400万円。スピードと実行力の実績[S6]
・「仮説→小さく試す→数字で確かめる」型で動けている[S6][S4]
・毎週1つ勝ちパターンを見つけ横展開＝毎日1%改善の姿勢[S4]
【懸念点】
・チームマネジメント経験が限定的（個人プレー偏重の可能性）
・希望年収が予算上限に近い／リファレンス未実施
【深掘り質問】
・自分と価値観の違うメンバーと成果を出した経験は？（カルチャーフィット）
・海外0→1の「勝ちパターン」は他人でも再現できる形に落ちているか？
・前職で経営陣と意見が割れた時、具体的に何をどう伝えたか？
【次アクション】
・現地代理店/前職上長へのリファレンスチェック
・二次面接でチーム適性を確認 → 問題なければ条件提示
```

**ブリーフィング**（抜粋イメージ）: 「モバイルへの追加投資を続けるべきか」
```
## 私の見解
続ける。ただし“無条件に”ではない。大胆なビジョンには長期の先行投資を厭わない[S2]が、
これは短期赤字を織り込み、資金と撤退基準を管理して踏み込む前提の投資だ[T2]。
## 想定されるリスク・反論
先行投資は短期の赤字を拡大させる（過去の学び）。KPIと撤退基準を先に。
## 会議で確認すべき論点
- 経済圏への貢献（送客・LTV）を数値で追えているか
- 何をもって「撤退」とするかの基準は明文化されているか
```

## 4. まとめ（このデモで見せられること）

1. **入力は議事録でも音声でもOK**（テキスト化してフォルダ投入）
2. **重要情報だけがシグナル化**され、因果ストーリーに整理される
3. **経営者の判断軸で**、即断・採用・壁打ち・ブリーフィングとして**根拠付きで**返る
4. **権限の線引き**（現場裁量か、社長確認か）まで示す
5. **採用判断**にも同じ思考を適用（合否の傾き＋評価点＋懸念＋深掘り質問。最終判断は人）

実在経営者でうまく再現できれば、自社の社長プロファイルに差し替えるだけで実運用に移せます。
