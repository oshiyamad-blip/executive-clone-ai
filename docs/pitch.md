# 経営者クローンAI — ピッチ資料

> 経営者の意思決定の仕組みと思考をデジタル化し、組織がいつでも壁打ちできる資産にする。

---

## 1. 課題

- **経営判断がボトルネック**: 部門長・経営企画は、経営者の時間が空くまで意思決定を待つ
- **会議が多い**: 事前のすり合わせ相手がおらず、確認のための会議が積み重なる
- **暗黙知が属人化**: 「なぜその判断をしたか」が経営者の頭の中にしか残らない

## 2. ソリューション

経営者の日常の言動を集約し、重要情報を **シグナル** として抽出、
過去の意思決定の文脈を **ストーリー** として整理。
これを基盤に **経営者クローン** と対話（壁打ち）できるようにする。

→ **会議前にAIと壁打ち** することで、会議の回数を削減し、意思決定の速度を上げる。
→ 経営者本人も気づいていない **行動の因果関係** を抽出し、経営の資産にする。

## 3. 仕組み（データフロー）

```
入力                         パイプライン                    蓄積            出力
─────────────────────────    ─────────────────────────────   ──────────    ─────────────────────
LINE（同期フォルダ）      ┐                                                 対話（Web / CLI）
Plaud録音（Zapier→Drive） ┤   収集 → 名寄せ → シグナル抽出                    会議前ブリーフィング
Gmail / Calendar / Meet   ┼→  → ストーリー構築（Claude）      → Notion  →   週次ダイジェスト
Google Docs               ┘   （日次・週次バッチ）                          （すべて根拠付き）
```

## 4. 差別化ポイント

- **根拠の明示**: 回答が「どのシグナル/ストーリーを根拠にしたか」を [S1][T1] で提示。ブラックボックスでない
- **Google Workspace ネイティブ**: Gmail / Calendar / Meet文字起こし / Docs をそのまま入力に
- **デバイス非依存**: Plaud NotePin S などウェアラブル録音をフォルダ投入で取り込み（機種を選ばない）
- **学習の循環**: AIとの対話ログも学習ソースとして還流
- **セキュア**: ZDR（学習非利用）API・アクセス制御・ローカル/プライベート実行

## 5. Google Workspace への導入

- **入力**: サービスアカウント＋ドメイン全体委任で、Gmail・Calendar・Meetの自動文字起こし・Docsを
  最小権限（readonly）で安全に取得。Workspaceの既存権限・監査の枠内で運用
- **出力**: 会議前ブリーフィング・週次ダイジェストを **Google Docs** として配布することも可能
- 追加のSaaSを増やさず、**既に使っているWorkspaceの中で完結**

## 6. 期待効果

- 会議回数の削減 / 意思決定の高速化
- 暗黙知（判断軸・成功/失敗パターン）の資産化
- 新任幹部のオンボーディング加速（「経営者ならどう考えるか」に即アクセス）

## 7. デモ（5分）

1. Web対話で「来期の投資配分は？」→ 経営者として**根拠付き**で回答
2. 会議前ブリーフィングを1コマンドで生成（見解＋根拠＋想定反論）
3. 参照元をクリック→ Notionの元データへ（透明性）

## 8. 現状とロードマップ

- **実装済み（現状）**: 入力〜出力の一気通貫
  - 入力: LINE / Plaud / Gmail / Calendar / Meet / Docs
  - 処理: 名寄せ・シグナル抽出・ストーリー構築（Claude、構造化出力）
  - 蓄積: Notion / 出力: 対話（Web/CLI）・ブリーフィング・週次ダイジェスト
  - 運用: 日次/週次バッチの自動化（cron/systemd）、デモモード
- **次の一手**:
  1. 参照制限（DBごとのアクセス層分離 / 要件4.1）
  2. RAG化（意味検索で関連データを引き、対話精度を向上）
  3. デバイス公式API連携（Limitless/Plaud Developer Platform）・LINE公式アカウント自動収集

## 9. クロージング
